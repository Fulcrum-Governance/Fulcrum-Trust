<objective>
Create the .github/workflows/publish.yml GitHub Actions workflow that automates PyPI publishing via OIDC Trusted Publishing. Three jobs: build (produces sdist + wheel artifact), publish-testpypi (uploads to TestPyPI first), publish-pypi (uploads to real PyPI after TestPyPI succeeds). Triggers on v* tag pushes.

Purpose: Manual uploads with API tokens are insecure and error-prone. PyPI Trusted Publishing (OIDC) eliminates stored secrets — the workflow authenticates via short-lived GitHub OIDC tokens that expire per-job. TestPyPI → PyPI serialization prevents publishing a broken package to the real index.

Output: .github/workflows/publish.yml — the automated publish pipeline ready to fire on tag push.
</objective>

Create .github/workflows/publish.yml with this exact content. Follow the official PyPA guide structure (source: https://packaging.python.org/en/latest/guides/publishing-package-distribution-releases-using-github-actions-ci-cd-workflows/).

Key rules from RESEARCH.md:
- `id-token: write` at JOB level (not workflow level) — minimum privilege
- Use `actions/upload-artifact@v4` and `actions/download-artifact@v4` (v3 is deprecated)
- Use `pypa/gh-action-pypi-publish@release/v1` (not @master — that branch is sunset)
- `needs: publish-testpypi` serializes TestPyPI before real PyPI
- No `username`, `password`, or `token` fields — OIDC handles authentication
- Environment names must match what gets configured in PyPI Trusted Publishing: `testpypi` and `pypi`

```yaml
name: Publish

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    name: Build distribution
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install build
        run: pip install build twine

      - name: Build distributions
        run: python -m build

      - name: Validate metadata
        run: twine check dist/*

      - uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/

  publish-testpypi:
    name: Publish to TestPyPI
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: testpypi
      url: https://test.pypi.org/p/fulcrum-trust
    permissions:
      id-token: write
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist/

      - uses: pypa/gh-action-pypi-publish@release/v1
        with:
          repository-url: https://test.pypi.org/legacy/

  publish-pypi:
    name: Publish to PyPI
    needs: publish-testpypi
    runs-on: ubuntu-latest
    environment:
      name: pypi
      url: https://pypi.org/p/fulcrum-trust
    permissions:
      id-token: write
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist/

      - uses: pypa/gh-action-pypi-publish@release/v1
```

Write the file exactly as shown. Do not add extra steps, do not add PYPI_TOKEN or similar secrets.

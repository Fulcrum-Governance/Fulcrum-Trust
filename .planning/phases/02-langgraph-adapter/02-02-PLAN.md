Append class `TestTrustAwareGraph` to tests/test_langgraph_adapter.py. Use `simple_builder` and `trust_manager` fixtures from Task 1.

**LANG-01: Zero-changes wrapping**

- `test_wrap_graph_runs_normally` — Wrap `simple_builder` with default TrustManager. `compile().invoke({"value": "start", "count": 0})` → assert result `count == 1` and `value == "done"`. This proves the graph runs correctly end-to-end after wrapping.

- `test_wrap_preserves_node_count` — Assert the wrapped+compiled graph produces the same final state as an unwrapped+compiled graph given identical inputs. Use two separate `simple_builder()` calls (pytest fixtures are per-test). Invoke both with `{"value": "", "count": 0}` and assert both results equal.

**LANG-02: Outcome classification integration**

- `test_trust_score_increases_on_success_nodes` — Fresh TrustManager. Before invoke: `score_before = tm.get_trust_score("a", "b")` (0.5). Invoke wrapped graph with `{"value": "start", "count": 0}`. After invoke: `score_after = tm.get_trust_score("a", "b")`. Assert `score_after > score_before` (successful node outputs improve trust). Use `agent_a="a", agent_b="b"`.

- `test_trust_decreases_on_error_node` — Build a graph with a single node that returns `{"error": "something failed"}`. Wrap it, invoke, assert `tm.get_trust_score("a", "b") < 0.5`. Note: use `builder.add_edge("__start__", "bad_node")` and `builder.add_edge("bad_node", END)`.

**LANG-03: Circuit-break routing**

- `test_circuit_break_fires_when_trust_below_threshold` — Pre-seed 3 FAILURE outcomes on `("a", "b")` into a TrustManager with `TrustConfig(threshold=0.5)` so `should_terminate()` is already True. Wrap `simple_builder` with this TrustManager. Register `on_circuit_break` callback. Invoke. Assert callback was called at least once.

- `test_no_circuit_break_on_healthy_trust` — Fresh TrustManager (trust starts at 0.5, threshold default 0.3 → no termination). Wrap simple graph. Register `on_circuit_break` callback. Invoke. Assert callback was NOT called (fired list remains empty).

**LANG-04: Callbacks**

- `test_on_trust_change_fires_per_node` — Wrap `simple_builder` (2 nodes). Register `on_trust_change`. Invoke. Assert callback fired exactly 2 times (once per node).

- `test_on_circuit_break_callback_receives_trust_state` — Pre-seed failures so trust below threshold. Register `on_circuit_break(lambda s: fired.append(s))`. Invoke. Assert `fired[0]` is a `TrustState` instance and `fired[0].trust_score < 0.5`.

- `test_on_recovery_fires_after_reset` — Pre-seed 3 FAILUREs with `TrustConfig(threshold=0.5)`. Track `_was_terminated` by: invoke once (circuit fires). Then call `tm.reset("a", "b")`. Register `on_recovery(lambda s: recovered.append(s))`. Invoke again (fresh builder, since compiled graphs are single-use). Assert recovered list has at least 1 entry.

**Async node support**

- `test_async_node_is_wrapped_correctly` — Create a StateGraph with an `async def` node:
  ```python
  async def async_node(state: SimpleState) -> dict:
      return {"count": state["count"] + 10}
  ```
  Wrap and compile. Invoke with `{"value": "", "count": 0}`. Assert result `count == 10`. Assert `tm.get_trust_score("a", "b") != 0.5` (trust was updated, proving wrapper ran).

**Missing langgraph guard**

- `test_import_error_without_langgraph` — Use `unittest.mock.patch` to temporarily make `_LANGGRAPH_AVAILABLE = False` on the `fulcrum_trust.adapters.langgraph` module. Assert that calling `TrustAwareGraph.__init__` raises `ImportError` containing the string `"pip install"`.
  ```python
  def test_import_error_without_langgraph(simple_builder, trust_manager):
      import fulcrum_trust.adapters.langgraph as adapter_mod
      original = adapter_mod._LANGGRAPH_AVAILABLE
      adapter_mod._LANGGRAPH_AVAILABLE = False
      try:
          with pytest.raises(ImportError, match="pip install"):
              TrustAwareGraph(simple_builder, trust_manager)
      finally:
          adapter_mod._LANGGRAPH_AVAILABLE = original
  ```
